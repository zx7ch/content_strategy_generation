## 1. Task Context

### Scope Boundary
- **Task ID**: 8.4
- **Task Name**: Phase 3: Xiaohongshu Page Capture Floating Panel
- **Phase**: XHS Extension MVP UX Optimization / Phase 3
- **Dependencies**: 8.2 active task runtime and 8.3 popup status panel are complete.
- **Task Goal**: Move the primary capture action into Xiaohongshu pages so users can capture where they are evaluating content, without opening popup as the normal path.

### In Scope
- Inject a fixed floating capture panel from `content.js`.
- Show active task state and visible note count in the panel.
- Show newly visible count after scrolling compared with the last successful capture.
- Trigger capture via background `CAPTURE_VISIBLE_PAGE`.
- Render success/failure feedback on the page.
- Preserve visible-content-only extraction.

### Out Of Scope
- Automatic scrolling, page turning, or simulated clicking.
- Workbench auto-refresh.
- Workbench auto-refresh.
- Multi-task switching or candidate preview inside the page.

### Required Deliverables
- Production: `extension/src/content.js` page panel implementation and small background compatibility if needed.
- Tests: static contract tests in `tests/unit/test_xhs_extension_mvp.py`.
- Spec/Docs: append Phase 3 implementation record to `spec.md`.

### Acceptance Criteria
- [ ] AC1 Content script injects `#xhs-mvp-capture-panel` on supported Xiaohongshu pages.
- [ ] AC2 Panel shows active task state and visible note count.
- [ ] AC3 Capture button sends `CAPTURE_VISIBLE_PAGE` to background and renders captured/new/duplicate result.
- [ ] AC4 Panel can be collapsed without removing page extraction behavior.
- [ ] AC5 Scrolling/visibility refresh updates visible and newly visible counts without auto-scrolling or clicking.
- [ ] AC6 Unsupported/no-task/server failure states show user-oriented page feedback.

## 2. Technical Design

### Module Structure
```text
experiments/xhs_extension_mvp/extension/src/content.js MODIFY
tests/unit/test_xhs_extension_mvp.py                   MODIFY
experiments/xhs_extension_mvp/spec.md                  MODIFY
```

### Flow
```text
content script loads
  -> requests active task
  -> injects floating panel
  -> refreshes visible count every 1500ms and after scroll
  -> user clicks capture
  -> content asks background CAPTURE_VISIBLE_PAGE
  -> background submits current visible extraction
  -> content renders result on page
```

### Constraints
- Use existing `extractCurrentPage()` and `extractSearchResultItems()` logic.
- Do not mutate source page content except the injected panel.
- Do not require popup for primary capture.
