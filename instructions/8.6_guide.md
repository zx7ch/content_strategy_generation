## 1. Task Context

### Scope Boundary
- **Task ID**: 8.6
- **Task Name**: Phase 5: Failure Handling and Runtime Stability
- **Phase**: XHS Extension MVP UX Optimization / Phase 5
- **Dependencies**: 8.2 active task runtime, 8.3 popup status panel, 8.4 page floating panel, and 8.5 workbench auto-refresh are complete.
- **Task Goal**: Make common extension/runtime failure modes explicit, recoverable, and user-readable without reintroducing the removed manual-token or popup-primary flow.

### In Scope
- Normalize background runtime failures with stable `code` values for server unavailable, no active task, unsupported page, content script unavailable, duplicate capture, invalid token, and submit failure.
- Make background proactively inject the content script and retry page status/extraction when the receiver is missing or stale.
- Keep tab-level capture concurrency protection through `activeCapturesByTab` and request-level dedupe through `request_id`.
- Update popup status copy so server, task, page, and helper failures give next-step guidance without mentioning fallback capture.
- Update the Xiaohongshu floating panel so users can retry task sync and open the workbench from no-task/server-down states.
- Add unit/static tests that lock the runtime failure contract and ensure old fallback/manual flow language stays removed.
- Update the spec implementation record for Phase 5 after validation.

### Out Of Scope
- Multiple active tasks or task switching UI.
- SSE or browser notifications.
- Automatic scrolling, pagination, or note clicking.
- Reintroducing any legacy manual token UI, `/mvp/captures`, or capture-token route.
- Real browser E2E automation; this phase validates code contracts with existing unit/static tests.

### Acceptance Criteria
- [ ] AC1 Server-unavailable failures surface as `server_unavailable` with clear retry guidance in popup and page panel.
- [ ] AC2 Missing active task failures surface as `no_active_task` with an `Open Workbench` recovery path.
- [ ] AC3 Non-Xiaohongshu/unsupported pages surface as `unsupported_page` and do not submit captures.
- [ ] AC4 Background attempts `ensureContentScript` and retries before surfacing `content_script_unavailable`.
- [ ] AC5 Repeated capture clicks for the same tab are rejected as `capture_already_running` without corrupting runtime state.
- [ ] AC6 Invalid capture token / failed submit paths map to typed user-facing failures and do not leak raw extension internals as the primary message.
- [ ] AC7 Static UI/runtime tests prove fallback capture and manual token language remain removed.

### Background Runtime
- Add a small `createRuntimeError(code, message)` helper and preserve `code` in `handleMessage` catch responses.
- `healthCheck` must map network and non-2xx health failures to `server_unavailable`.
- `syncActiveTask` keeps returning `ok: true` when the server has no active task, because no-task is a valid product state.
- `captureVisiblePage` must:
  - resync active task before capture;
  - return `no_active_task` if no runtime token exists;
  - block same-tab concurrency with `capture_already_running`;
  - attempt `ensureContentScript` and retry extraction if the tab receiver is missing;
  - map unsupported extraction to `unsupported_page`;
  - map submit 401 to `capture_token_invalid`, 403 to `capture_task_mismatch`, and other non-2xx responses to `capture_submit_failed`.
- `getPageStatus` must attempt content-script injection and retry before returning `content_script_unavailable`.
- Add `OPEN_WORKBENCH` so page panel recovery can open the current server workbench without duplicating server URL logic in content script.

### Popup Status Panel
- Preserve status-panel-only behavior.
- Map `server_unavailable`, `no_active_task`, `unsupported_page`, and `content_script_unavailable` to clear next-step messages.
- Remove all fallback-capture wording.

### Page Floating Panel
- Add secondary buttons for `重试连接/同步` and `打开工作台`.
- When active task sync fails, set `server_unavailable` or `runtime_error` feedback visibly instead of only logging.
- `toPanelMessage` must map the exact runtime codes from this guide.
- Unsupported pages keep the capture button disabled and show guidance to open a Xiaohongshu search or note page.
