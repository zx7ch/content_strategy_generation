## 2. Technical Design

### Module Structure
```text
tests/unit/test_xhs_extension_mvp.py MODIFY
experiments/xhs_extension_mvp/spec.md MODIFY
```

### Acceptance Test Shape
- Prefer a high-signal test that exercises the real local server app with deterministic in-memory/tmp SQLite state.
- Reuse the extension capture endpoint and `X-Capture-Token` header; do not call deleted routes except to assert 404.
- Keep frontend assertions static and contract-based because browser automation is out of scope for this phase.
- Avoid duplicating every unit test; combine the final user path into an acceptance-style scenario.
