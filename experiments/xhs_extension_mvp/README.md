# XHS Extension MVP

独立验证版原型，用来测试这条链路是否真实可用：

`浏览器插件 + 用户触发采集 + deterministic expand query + 本地候选生成`

这套 MVP 与当前正式服务隔离：

- 不接 `app/api/routes/router.py`
- 不接现有 `Session/Job/Worker/LangGraph`
- 不写正式 `data/xhs_agent.db`

## 目录

- `server/`: 独立 FastAPI 服务
- `web/`: 单页工作台
- `extension/`: Chrome MV3 插件

## 适用范围

- 只支持 Chrome 桌面版
- 只采集当前页面可见内容
- 不自动滚动、不翻页、不模拟点击
- 只验证“能不能用”，不验证长期稳定性和规模化

## 第一次安装与启动

第一次使用时，按下面顺序做一遍即可。

### 1. 进入项目目录

先进入这个仓库根目录，也就是包含 `app/`、`experiments/`、`setup_env.sh` 的目录：

```bash
cd /Users/czx/Documents/agentic/xhs_note_generator
```

如果你的本地路径不同，把上面的路径替换成你自己的仓库路径即可。

### 2. 初始化并激活虚拟环境

第一次使用时，先在仓库根目录执行：

```bash
./setup_env.sh
source .venv/bin/activate
```

说明：

- `./setup_env.sh` 会创建 `.venv`、安装 `requirements.txt`，并在缺少 `.env` 时自动从 `.env.example` 复制
- `source .venv/bin/activate` 会把当前终端切换到这个项目自己的 Python 环境
- 如果你关闭终端后重新打开，需要再次执行一次 `source .venv/bin/activate`

### 3. 启动 MVP 服务

在仓库根目录执行：

```bash
uvicorn experiments.xhs_extension_mvp.server.app:app --host 127.0.0.1 --port 8010
```

打开工作台：

```text
http://127.0.0.1:8010/
```

首次运行会创建：

```text
data/xhs_extension_mvp.db
data/xhs_extension_mvp.log
```

### 4. 在 Chrome 安装插件

1. 打开 Chrome，进入 `chrome://extensions/`
2. 打开右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择目录：

```text
experiments/xhs_extension_mvp/extension
```

5. 确认扩展列表里出现 `XHS Extension MVP`

这一步只需要做一次。后续如果只是正常使用，不需要每次重新选择目录。

### 5. 首次采集完整流程

1. 在工作台输入搜索主题，点击 `创建任务`
2. 创建任务成功后，系统会把该任务设为当前 active task，插件会自动同步任务上下文
3. 如果系统生成的 `拓展搜索` 不够贴合，你可以先在 `自定义拓展词` 输入框里自己补充搜索词，每行一条，点击 `添加自定义拓展词`
4. 在工作台右侧 `拓展搜索` 里选一个搜索词，点 `打开小红书搜索`
5. `热点追踪区域` 会在创建任务后基于你的 `原始搜索词` 自动刷新一次；后续也可以手动点击 `更新热点`
6. 小红书页面右下角会出现 `XHS 采集助手`
7. 在页面浮层里点击 `采集当前页`
8. 工作台会自动检测采集版本变化并刷新候选方向
9. 在 `候选方向` 区域查看推荐理由、可写角度、证据概览和代表样本

## 日常使用步骤

后续日常使用时，一般只需要重复下面这条链路：

1. 进入仓库根目录
2. 如果当前终端还没激活虚拟环境，先执行 `source .venv/bin/activate`
3. 启动 MVP 服务
4. 打开工作台 `http://127.0.0.1:8010/`
5. 创建任务
6. 如有需要，先补充 `自定义拓展词`
7. 打开一个小红书搜索页或笔记详情页
8. 在页面右下角的 `XHS 采集助手` 里点击 `采集当前页`
9. 回到工作台查看自动刷新的采集结果和候选方向
10. 如需查看当前原始搜索词下的真实热门结果，可点击 `更新热点`

如果你在小红书页面里继续向下滚动，页面上出现了新的可见笔记，可以再次点击页面浮层里的 `采集当前页`。系统会自动合并去重。

## 更新后如何继续使用

代码更新后，不同类型的改动处理方式不一样。为了稳妥，建议按下面顺序检查。

### 情况 A：只改了服务端或工作台页面

比如改了：

- `server/`
- `web/`
- README 或其他非插件文件

需要做的事：

1. 进入仓库根目录
2. 如果当前终端还没激活虚拟环境，先执行：

```bash
source .venv/bin/activate
```

3. 停掉旧的 `uvicorn` 进程
4. 重新执行：

```bash
uvicorn experiments.xhs_extension_mvp.server.app:app --host 127.0.0.1 --port 8010
```

5. 回到浏览器刷新工作台页面

这种情况通常不需要重新安装插件。

### 情况 B：改了插件代码

比如改了：

- `extension/manifest.json`
- `extension/src/popup.html`
- `extension/src/popup.js`
- `extension/src/content.js`

需要做的事：

1. 确认最新代码已经在本地
2. 打开 `chrome://extensions/`
3. 找到 `XHS Extension MVP`
4. 点击扩展卡片上的 `重新加载`
5. 回到目标小红书页面，手动刷新一次该页面
6. 重新打开插件 popup 再执行采集

这里最容易漏掉的是第 5 步。因为插件重新加载后，当前已经打开的小红书页面里的 content script 可能还是旧的，刷新页面后才会重新挂载。

### 情况 C：插件异常，想彻底重新安装

如果你点击 `重新加载` 后仍然不正常，或者扩展目录发生了较大变化，可以完整重装一次：

1. 打开 `chrome://extensions/`
2. 删除当前的 `XHS Extension MVP`
3. 点击 `加载已解压的扩展程序`
4. 重新选择：

```text
experiments/xhs_extension_mvp/extension
```

5. 打开任意一个小红书页面并手动刷新一次
6. 再回到工作台重新生成 token 并开始采集

### 情况 D：更新后依赖或 Python 环境也变了

如果这次更新包含了 Python 依赖变化，或者你的 `.venv` 被删掉了，建议在仓库根目录重新执行：

```bash
./setup_env.sh
source .venv/bin/activate
```

然后再启动 MVP 服务和继续使用插件。

## 常见问题

### popup 提示 `Could not establish connection. Receiving end does not exist.`

- 当前标签页里的 content script 还没有挂上
- 这通常发生在你刚刚 `重新加载` 了插件，但目标小红书页面没有刷新
- 新方案的主入口是小红书页面里的 `XHS 采集助手`
- 如果浮层没有出现，手动刷新一次小红书页面，让 content script 重新挂载

### 工作台里没有候选

- 先确认你已经在小红书页面浮层里成功点击过一次 `采集当前页`
- 工作台会自动监听采集版本变化并刷新候选方向，通常不需要手动操作
- 如果搜索页当前可见内容太少，可以继续往下滚动后再次采集
- 如果系统给的 `拓展搜索` 不够贴切，可以先自己补几条 `自定义拓展词` 再去搜
- 也可以先用 `手动补充` 输入链接或文本线索做降级验证

### 热点榜和拓展搜索为什么不是一回事

- 这是当前实现的预期行为
- `拓展搜索` 是为了帮你打开更多搜索入口
- `热点榜` 只基于你输入的 `原始搜索词`
- 热点榜直接调用 spider 的真实排序接口，分别拉取 `最多点赞`、`最多评论`、`最多收藏`
- 创建任务后会自动刷新一次，后续也可以手动点击 `更新热点`

## 自定义拓展词

如果系统生成的 `拓展搜索` 不够符合你的搜索习惯，可以在工作台直接添加 `自定义拓展词`：

- 输入位置就在 `拓展搜索` 区块顶部
- 每行输入一条搜索词
- 点击 `添加自定义拓展词` 后，会和系统生成的词一起显示
- 自定义拓展词可以直接点开搜索
- 重复的词不会重复添加
- 自定义拓展词支持单独删除；系统自动生成的词不能删除
- 自定义拓展词不会参与 `热点榜` 抓取

适合用在这些场景：

- 你已经知道平台上更容易搜到结果的口语化词
- 系统生成的搜索词还不够贴题
- 你想补一些非常具体的人群词、场景词、品牌词或问题词

## 手动补充

如果插件不可用，可以在工作台使用 `手动补充`：

- 每行粘贴一个链接，或一条文本线索
- 点击 `导入手动补充`
- 这些内容会进入同一套候选生成逻辑

注意：

- `自定义拓展词` 是为了补搜索词，帮助你去搜更合适的结果
- `手动补充` 是为了直接导入链接或文本线索，跳过插件采集做降级验证

## 热点追踪区域

热点榜的当前实现是：

- 创建任务后自动刷新一次
- 后续可以手动点击 `更新热点`
- 只基于用户输入的 `原始搜索词`
- 直接调用 spider 的真实排序接口，分别拉取：
  - `最多点赞`
  - `最多评论`
  - `最多收藏`
- 如果刷新失败，会保留上一次成功结果

这意味着：

- 热点榜更接近“当前这个搜索词在平台上的真实热门结果”
- 它不会和 `默认拓展词` / `自定义拓展词` 保持完全一致
- 热点刷新通常会比本地 UI 操作慢，因为它会真实请求 spider 搜索接口，而不是读取本地缓存

## 调试日志

- 服务端会把结构化日志同时输出到终端和 `data/xhs_extension_mvp.log`
- 可通过环境变量覆盖日志路径：

```bash
export XHS_EXTENSION_MVP_LOG_PATH=/tmp/xhs_extension_mvp.log
```

- 插件侧调试日志会输出到：
  - 当前小红书页面的 DevTools Console
  - 插件 popup 的 DevTools Console
